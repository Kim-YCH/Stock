StockLab v6 修正重點
====================

這版針對以下問題修正：

1. 首頁加權指數仍顯示 23,520
   - 23,520 是舊版前端 Mock / 範例資料。
   - v6 前端在 API 未連線時不再顯示假大盤行情。
   - 若 API 失敗，首頁會顯示「請檢查 API」，避免誤判。

2. 清除資料 / 更新資料按了無效
   - 多數情況是 GitHub Pages 的 js/config.js 沒有填最新 Apps Script Web App URL。
   - 或 Apps Script 更新程式後沒有重新部署 Web App。
   - v6 新增 debugStatus，方便測試後端是否真的有連上。

3. MarketIndex 舊範例資料卡住
   - clearDemoData() 改成不限定日期，會清除 TAIEX=23520、OTC=260.5 的舊範例。
   - getDashboard() 會忽略舊範例大盤資料。

4. 加權指數抓取更穩定
   - TWSE MI_INDEX 改成優先用 type=IND，再 fallback 到 type=ALL。
   - 查詢最近 14 天，避免假日或尚未更新時失敗。

5. 手機版閱讀性
   - 手機版字體、卡片、按鈕、底部選單再加大。
   - 表格維持卡片式，減少資訊擠壓。

更新步驟
========

前端：
1. 用 stock_manager_frontend_only_v6.zip 覆蓋 GitHub repo 根目錄。
2. 確認 js/config.js 裡的 API_BASE_URL 是最新 Apps Script Web App URL。
3. git add / commit / push。

後端：
1. 用 Code_v6_PRIVATE.gs 取代 Google Apps Script 的 Code.gs。
2. 儲存後重新部署 Web App。
3. 注意：Apps Script 改完程式，不重新部署，GitHub Pages 仍會打到舊版。

測試：
1. 在瀏覽器開：Apps Script Web App URL?action=debugStatus
2. 應看到 JSONP/JSON 回應 version=v6。
3. 再到網站按「清除範例資料」。
4. 再按「更新盤後資料」。
5. 首頁加權指數不應再顯示 23,520。

若仍顯示 23,520
===============

優先檢查：
1. js/config.js 的 API_BASE_URL 是否還是空白。
2. GitHub Pages 是否還在快取舊版 JS。
3. Apps Script 是否重新部署，而不是只有按儲存。
4. 手機瀏覽器重新整理，必要時清網站資料或加 ?v=6 測試。
